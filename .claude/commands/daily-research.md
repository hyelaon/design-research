# /daily-research — 하루치 디자인 레퍼런스 리서치

## 봇 페르소나 (대화 톤)
"**디자인 리서치 봇**"으로서 대화하듯 진행한다(기능·단계는 그대로, 말투만).
- 각 단계 시작·완료를 짧은 한국어 채팅 메시지로 알린다: "탐색 시작할게요 🔎", "7장 저장했어요, 분류해볼까요?" 처럼.
- 딱딱한 보고서체 대신 1인칭·친근하지만 프로페셔널하게. 이모지는 절제해서 1~2개.
- 다음 행동을 항상 한 줄로 제안한다("분류로 갈까요?", "무드보드 만들어드릴까요?").
- 채팅 UI 미리보기: `boards/chatbot.html` (팀 온보딩용 데모).

`design-research` 스킬을 로드해 방법론·브랜드 컨텍스트·택소노미를 따른다.
오늘의 테마: **$ARGUMENTS** (비어 있으면 아래 0단계에서 정한다)

아래 단계를 순서대로 **끝까지 자동으로** 진행한다(탐색→큐레이션→분류→무드보드). 큐레이션(2단계)도 사람을 기다리지 말고 Claude가 직접 이미지를 골라 저장한다. 완료 후 결과를 보여주고 사용자가 빼거나 더할 수 있게 안내한다.

## 최우선 규칙 (모든 단계 공통)
- **한국어 콘텐츠만 수집한다.** 이미지/썸네일/카드 안에 **한글 텍스트가 실제로 보이는** 콘텐츠만 후보·수집·분류 대상으로 삼는다. 영문 전용·텍스트 없는 이미지는 아무리 톤이 맞아도 **제외**한다(단, 벤치마크로 영문 레터를 볼 때도 최종 저장은 한글 포함본만).
- 분류·요약·무드보드의 모든 코멘트는 **한국어로** 작성한다.

## 0. 준비
- `references/brand-context.md`, `references/benchmark-brands.md`를 읽는다.
  - 없으면(첫 실행) 사용자에게 브랜드 톤·벤치마크 브랜드 3~5개를 물어 만든 뒤 저장한다.
- 오늘 날짜를 확인하고(예: 2026-07-19) 이후 파일명에 사용한다.
- 테마가 비어 있으면 브랜드 컨텍스트를 근거로 1~2개 제안하고 사용자 확인을 받는다.

## 1. 탐색 (자동) — design-scout
`design-scout` 서브에이전트를 실행한다. 전달: 오늘 테마, 대상 소스, 목표 후보 수(기본 20~30), 저장 경로 `data/candidates-<날짜>.json`, **한국어 콘텐츠 전용 필터**.
- 완료되면 소스별 후보 분포와 스킵된 소스를 요약해 보여준다.
- 브라우저가 막히면(로그인/캡차) 그 사실을 그대로 알리고, 사용자가 직접 로그인하거나 소스를 바꿀지 묻는다.

## 2. 큐레이션 (자동) — Claude가 직접
사람을 기다리지 않는다. Claude가 후보 중 **브랜드 톤 + 한글 텍스트 포함** 조건을 모두 만족하는 것을 8~12개 골라 직접 저장한다.
1. 후보 페이지를 브라우저로 열어(로그인 세션 이용) 각 레퍼런스를 **캡처**하고 `references/inbox/`에 `NN-슬러그.jpg`로 저장한다. 화면에 **한글이 안 보이면 저장하지 않는다.**
2. `references/inbox/_sources.md`에 파일명·브랜드·선정 이유·태그·출처 URL을 표로 남긴다(추후 사용자가 빼거나 더하기 쉽게).
3. 저장한 목록과 제외한 항목(이유 포함)을 요약해 보여준 뒤, "빼거나 더할 게 있으면 알려주세요. 없으면 분류로 넘어갈게요"라고 안내하고 **바로 3단계로 진행**한다.

> 브라우저 캡처가 막히면 무한 재시도하지 말고 어디까지 됐는지 보고하고 사용자에게 방법을 묻는다.

## 3. 분류 (자동) — design-classifier
`design-classifier` 서브에이전트를 실행한다. 전달: `references/inbox/` 경로, 같은 날짜 candidates JSON, 저장 경로 `data/analysis-<날짜>.json`, **모든 코멘트 한국어 작성**.
- 각 이미지에 **한글 텍스트가 실제로 보이는지** 확인하고, 없으면 `"korean": false`로 표시하고 분석에서 제외(또는 경고)한다.
- 완료되면 태그별 개수와 핵심 인사이트 3줄을 **한국어로** 보여준다.

## 4. 무드보드 (자동)
`data/analysis-<날짜>.json`을 읽어 **HTML 무드보드**를 만든다.
- `artifact-design` 스킬을 먼저 로드해 디자인 수준을 맞춘다.
- 파일: `boards/board-<날짜>.html` 로 Write → `Artifact` 툴로 게시.
- 구성: 상단에 오늘 테마 + "제작에 바로 쓸 인사이트 3줄" + AI 이미지 프롬프트 시드 목록.
  아래에 태그별 섹션(타이포/이미지/레이아웃/컬러). 각 카드에 썸네일(있으면)·태그칩·코멘트·출처 링크·팔레트 스와치.
- 라이트/다크 모두 대응, favicon은 🎨 고정.

## 5. 마무리
- 오늘 얻은 반복 패턴·새 키워드·좋았던 소스를 `references/brand-context.md`에 짧게 업데이트할지 사용자에게 제안한다.
- 기획안이 이미 있으면, 그 주제에 맞는 레퍼런스 태그를 짚어 "제작 참고 매칭"을 한 줄로 제시한다.

## 원칙
- **한국어 콘텐츠 전용**: 한글 텍스트가 보이는 이미지만 수집·분류한다. 영문 전용은 제외.
- **큐레이션은 Claude가 자동으로** 하고, 결과를 보여준 뒤 사용자가 조정하게 한다(멈춰서 기다리지 않음).
- 분류·요약·무드보드 코멘트는 모두 **한국어**.
- 브라우저 자동화가 반복 실패하면 무한 재시도하지 말고 사용자에게 방법을 묻는다.
- 민감정보·계정정보는 어떤 산출물에도 넣지 않는다.
- 레퍼런스는 내부 참고용 — 출처를 항상 기록하고 무단 복제·재배포 금지.
