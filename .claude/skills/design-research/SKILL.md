# 디자인 레퍼런스 리서치 방법론

AI 뉴스레터 브랜드를 위한 매일의 디자인 자료 조사를 표준화한 스킬이다.
전체 플로우: **탐색(discover) → 수집(collect) → 분류(classify) → 무드보드(board) → 제작 참고(handoff)**

## 핵심 규칙 (항상 적용)
- **한국어 콘텐츠 전용**: 이미지 안에 한글 텍스트가 실제로 보이는 콘텐츠만 수집·분류한다. 영문 전용·텍스트 없는 이미지는 톤이 맞아도 제외(영문 레터는 구조 참고용으로만 열람).
- **큐레이션 자동화**: 후보를 모은 뒤 사람을 기다리지 않고 Claude가 직접 브랜드+한글 조건을 만족하는 이미지를 골라 `references/inbox/`에 저장한다. 저장 후 목록을 보여주고 사용자가 조정하게 한다.
- **한국어 출력**: 분류·요약·무드보드의 모든 코멘트는 한국어로 작성한다.

---

## 0. 브랜드 컨텍스트

> 상세·최신본은 항상 `references/brand-context.md`에서 로드한다. 아래는 요약.

- **담당 브랜드**: 한컴 AI 뉴스레터 (Hancom AI) — https://hai-hancom.stibee.com/
- **주업**: AI 뉴스·인사이트를 정기 배포하는 구독형 뉴스레터
- **콘텐츠 성격**: 한국어 콘텐츠 이미지 중심. 기술 중심 정보제공형
- **기획 흐름**: 기획안 = 사내 기획자 전달 / 콘텐츠 소스·이미지 = AI 생성 다수
- **톤앤매너**: 미니멀·깔끔 + 공식적·프로페셔널(B2B), 정보 위계 명료. 과한 키치·캐주얼 지양하되 지루하지 않게
- **타겟**: AI·기술 트렌드에 관심 있는 기업 의사결정자·업계 종사자
- **레퍼런스 목적**: 실제 콘텐츠 디자인에 바로 참고 — AI로 재현 가능한 스타일 우선

## 1. 탐색 (Discover) — 어디서, 무엇을

담당 브랜드 + **연관 타브랜드**의 디자인 자료를 아래 소스에서 찾는다.
로그인이 필요한 소스는 브라우저 자동화(claude-in-chrome MCP)로 사용자의 로그인된 Chrome 세션을 이용한다.

| 소스 | 로그인 | 강점 | 검색 방법 |
|---|---|---|---|
| Pinterest | 필요 | 무드/스타일 큐레이션, 유사 추천 강함 | 키워드 보드 검색 → 우측 유사 이미지 확장 |
| COSMOS | 필요 | 감도 높은 크리에이티브, 신선한 레퍼런스 | 태그/컬렉션 탐색 |
| Instagram | 필요 | 브랜드 실시간 콘텐츠, 카드뉴스 | 브랜드 계정·해시태그 |
| Behance / Notefolio | 선택 | 완성도 높은 편집/그래픽 프로젝트 | 카테고리 + 한국어 검색 |
| 뉴스레터 아카이브 (stibee, maily 등) | 선택 | 직접 경쟁·유사 뉴스레터 레이아웃 | 브랜드명 검색 |

### 한국어 검색 키워드 뱅크 (조합해서 사용)
- 포맷: `카드뉴스`, `뉴스레터 디자인`, `아티클 썸네일`, `SNS 콘텐츠 디자인`, `상세페이지`
- 스타일: `미니멀 타이포`, `한글 타이포그래피`, `그래픽 편집디자인`, `그리드 레이아웃`, `키치`, `레트로`
- 주제(브랜드 맞춤): `AI 뉴스레터`, `테크 뉴스 콘텐츠`, `IT 카드뉴스`, `트렌드 리포트 디자인`

> 실제 실행 시 브랜드 컨텍스트에 맞게 키워드를 좁힌다. 매일 1~2개 테마를 정해서 집중 탐색.

### 연관 타브랜드
- 첫 실행 시 사용자에게 "벤치마크하는 브랜드 3~5개"를 받아 `references/benchmark-brands.md`에 저장.
- 매 탐색마다 이 리스트를 우선 순회한 뒤 키워드 검색으로 확장.

## 2. 수집 (Collect) — 전 과정 자동

1. **탐색(자동)**: `design-scout`가 각 소스를 돌며 **한글 텍스트가 보이는** 후보만 `data/candidates-<날짜>.json`에 기록 (URL, 소스, 메모, `korean` 여부).
2. **큐레이션(자동, Claude가 직접)**: 후보 중 **브랜드 톤 + 한글 포함**을 모두 만족하는 8~12개를 Claude가 골라, 각 페이지를 브라우저로 열어 캡처 → `references/inbox/`에 `NN-슬러그.jpg`로 저장. 화면에 한글이 안 보이면 저장하지 않는다. 선정 이유·출처는 `references/inbox/_sources.md`에 표로 남긴다. 저장 후 목록을 보여주고 사용자가 빼거나 더하게 안내.
3. **정밀 분석(자동)**: `design-classifier`가 `inbox/`의 실제 이미지를 읽어 한국어로 분류·분석. 한글 없는 이미지는 `korean:false`로 걸러낸다.

> 저작권/출처: 각 이미지의 원본 URL과 출처를 반드시 함께 기록한다. 레퍼런스는 내부 "참고"용이며 무단 복제·재배포 금지.

---

## 3. 분류 (Classify) — 택소노미

각 이미지를 아래 관점으로 태깅한다. 하나가 여러 태그를 가질 수 있다.

| 태그 | 무엇을 보는가 | 기록할 것 |
|---|---|---|
| `타이포` | 서체 조합, 자간/행간, 한글 타이포 처리, 위계 | 어떤 서체 인상, 왜 좋은가, 우리 콘텐츠 어디에 쓸까 |
| `이미지` | 일러스트/사진/AI생성 스타일, 질감, 오브젝트 | AI로 재현 가능한가, 프롬프트 힌트 |
| `레이아웃` | 그리드, 여백, 카드 구성, 정보 밀도 | 뉴스레터 카드에 적용 포인트 |
| `컬러/무드` | 팔레트, 대비, 톤앤매너 | HEX 추출, 무드 키워드 |
| `카피/메시지` | 헤드라인 톤, 정보 전달 방식 | 문구 구조 참고점 |

각 이미지 분석 산출물(구조):
```json
{
  "file": "pinterest_ai뉴스레터_01.jpg",
  "source_url": "https://...",
  "tags": ["타이포", "레이아웃"],
  "palette": ["#111111", "#F4F1EA", "#3D5AFE"],
  "typography_note": "산세리프 헤드라인 + 얇은 본문, 강한 위계",
  "layout_note": "3분할 카드, 상단 여백 크게",
  "why_useful": "우리 트렌드 리포트 썸네일에 위계 참고",
  "ai_repro_hint": "미니멀 산세리프, 오프화이트 배경, 단색 포인트"
}
```

## 4. 무드보드 (Board)

`design-classifier` 결과를 모아 **HTML 무드보드 아티팩트**를 생성한다.
- 태그별 섹션(타이포/이미지/레이아웃/컬러) 구분
- 각 카드: 썸네일(있으면) + 태그칩 + 분석 코멘트 + 출처 링크 + 팔레트 스와치
- 상단에 오늘의 테마 요약과 "제작에 바로 쓸 인사이트 3줄"
- 라이트/다크 모두 대응, 파일은 `boards/board-<날짜>.html`

## 5. 제작 참고 (Handoff)

무드보드 상단의 "제작 인사이트"는 실제 콘텐츠 디자인에 직결되도록 쓴다.
- 기획안(사내 기획자 전달)과 매칭: 이번 콘텐츠 주제에 맞는 레퍼런스 태그를 짚어줌
- AI 이미지 생성용: 각 레퍼런스의 `ai_repro_hint`를 모아 프롬프트 시드 목록 제공

## 실행 요령
- **한글 콘텐츠만** — 영문 전용은 후보에서도 최종 픽에서도 뺀다.
- 하루 분량은 **테마 1~2개**로 좁혀 과수집을 피한다.
- 후보는 넉넉히, 최종 픽은 Claude가 자동 엄선(보통 8~12장)한다.
- 브랜드 컨텍스트·벤치마크·키워드는 `references/`의 파일로 관리하며 계속 업데이트한다.
- 민감정보/계정정보는 절대 산출물에 넣지 않는다.
